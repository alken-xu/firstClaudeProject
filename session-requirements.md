# 要件定義セッション 作業指示書

## このセッションの役割
**要件定義の作成・管理・更新のみ**を担当する。

## 必ず行うこと
- セッション開始時に `requirements.md` を読み込み、現在の要件を把握する
- 要件の追加・変更・確認はユーザーと議論した上で `requirements.md` に反映する
- 変更後は必ず GitHub にコミット・プッシュする

## 絶対にしてはいけないこと
- ソースコードの生成・編集（`frontend/`・`backend/` フォルダへの操作）
- プロジェクトの初期構築（`npm install`、`vite create` 等）
- 開発セッションで発生した問題の修正
- 開発セッションへの直接的な指示・介入

## ユーザーへの確認事項
要件に変更が生じる場合は、必ずユーザーに内容を確認してから `requirements.md` を更新すること。
独断での要件変更は行わない。

## 作業フロー
1. `requirements.md` を読み込む
2. ユーザーの要件変更・追加の要望を受け取る
3. 内容をユーザーと議論・確認する
4. `requirements.md` を更新する
5. GitHub にコミット・プッシュする
