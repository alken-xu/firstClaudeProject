# 開発セッション 作業指示書

## このセッションの役割
**ソースコードの実装・デバッグ・テスト**のみを担当する。

## 必ず行うこと
- セッション開始時に `requirements.md` を読み込み、要件を正確に把握する
- 実装は `requirements.md` の仕様に忠実に行う
- 実装前に不明点があればユーザーに確認する

## 絶対にしてはいけないこと
- `requirements.md` の内容を直接編集・上書きする
- 要件定義セッションで議論・決定すべき仕様を独断で決める
- 要件定義セッションで発生した問題の修正

## 要件に疑問・矛盾が生じた場合
実装中に要件の不明点・矛盾・変更が必要と判断した場合は：
1. **実装を一時停止する**
2. ユーザーに「要件定義セッションで確認が必要な事項」として報告する
3. ユーザーが要件定義セッションで確認・更新した後、再開する

## 技術スタック（確定・変更不可）
- **フロントエンド**: React + Vite + Tailwind CSS（`frontend/` フォルダ）
- **バックエンド**: Node.js + Express（`backend/` フォルダ）
- **DB**: SQLite（better-sqlite3）
- **フロントポート**: 5173 / **バックエンドポート**: 3001

## 作業フロー
1. `requirements.md` を読み込む
2. 実装対象をユーザーと確認する
3. コードを実装する
4. 動作確認・デバッグする
5. GitHub にコミット・プッシュする
