# テストセッション指示書

**作成日**: 2026-04-05  
**セッション種別**: 自動単体テストセッション

---

## 1. セッション目的

Playwright を使った自動単体テストを実施し、テストケース定義・実施エビデンス（スクリーンショット）・結果を **単一の Excel ファイル** にまとめる。

---

## 2. テスト対象

| サーバー | URL |
|---|---|
| フロントエンド | http://localhost:5173 |
| バックエンド API | http://localhost:3001/api |

---

## 3. テスト基盤構成

```
tests/
├── package.json          ← 依存定義（playwright + exceljs）
├── run-tests.js          ← テストランナー本体（テストケース定義 + Excel生成）
├── screenshots/          ← 各テストのスクリーンショット（実行時に自動生成）
│   ├── TC-001.png
│   ├── TC-002.png
│   └── ...
└── test-results.xlsx     ← テスト結果（実行時に自動生成）
```

## 5. Excel 出力仕様

### Sheet 1: テストケース一覧

| 列 | 内容 |
|---|---|
| No. | 連番 |
| テストID | TC-001 など |
| カテゴリ | ページ表示 / 予約フロー / フォーム |
| テスト名 | テストの概要 |
| 前提条件 | 実行前の状態 |
| 操作手順 | 実施ステップ |
| 期待結果 | 想定される結果 |
| テスト結果 | PASS / FAIL / ERROR（色分け） |
| エラー内容 | FAIL / ERROR 時のメッセージ |
| エビデンスシート | 対応するシート名 |
| 実施日時 | 実行時刻 |
| 実施時間(ms) | 処理時間 |

- 結果は色分け（PASS=薄緑・FAIL=薄赤・ERROR=薄黄）
- ヘッダー固定（ウィンドウ枠の固定）

### Sheet TC-001 〜 TC-016: 個別エビデンスシート

各シートに以下を記載：
1. テストタイトル行（深緑背景）
2. テスト結果バッジ（PASS/FAIL/ERROR 色付き）
3. テスト情報テーブル（カテゴリ・前提条件・手順・期待結果・エラー内容）
4. スクリーンショット（1280×900px、テスト時点の画面）

## 6. テストの実行方法

### 前提条件
- フロントエンドサーバーが `http://localhost:5173` で起動していること
- バックエンドサーバーが `http://localhost:3001` で起動していること

### 初回セットアップ（初回のみ）
```bash
cd tests
npm install
npx playwright install chromium
```

### テスト実行
```bash
cd tests
node run-tests.js
```

### 出力確認
- `tests/test-results.xlsx` — テスト結果 Excel ファイル
- `tests/screenshots/TC-XXX.png` — 各テストのスクリーンショット
